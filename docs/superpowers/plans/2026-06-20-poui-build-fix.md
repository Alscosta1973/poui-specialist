# PO-UI Build Fix — Self-Correction Loop

Compila o projeto Angular após geração de componentes, detecta erros TypeScript/template e os corrige automaticamente nos arquivos gerados pelo plugin (até 3 tentativas).

> Esta skill é invocada automaticamente pelo `/poui-specialist:generate` e `/poui-specialist:generate-batch`. Não é necessário chamá-la manualmente.

---

## Passo 1 — Localizar raiz do projeto Angular

Buscar `angular.json` subindo até 3 níveis a partir do diretório atual:

```powershell
Get-ChildItem -Path . -Filter angular.json -Recurse -Depth 3 | Select-Object -First 1 FullName
```

**Se não encontrado:** exibir e encerrar:

```
⚠ Nenhum projeto Angular encontrado — build verification ignorada.
Os arquivos foram gerados mas não foram compilados.
```

---

## Passo 2 — Compilar o projeto

Na raiz do projeto Angular, rodar:

```powershell
$buildOutput = & ng build --configuration development 2>&1
$buildSuccess = $LASTEXITCODE -eq 0
```

Timeout: 120 segundos. Se timeout estourar, exibir:

```
⚠ Build demorou mais de 120s — verifique se o projeto tem erros de dependência.
```

---

## Passo 3 — Inspecionar resultado

**Se build passou (exit code 0):**

```
## Build Verification

✅ Build passou na tentativa 1 — nenhum erro TypeScript ou de template.
   Arquivos gerados compilam sem erros.
```

Encerrar.

**Se build falhou:** parsear o output procurando linhas no formato:

```
ERROR in <caminho-do-arquivo>:<linha>:<coluna>
<mensagem do erro>
```

Agrupar todos os erros por arquivo. Separar em dois grupos:
- **Arquivos do plugin:** contêm a linha `@generated  poui-specialist` → corrigir no Passo 4
- **Arquivos pré-existentes:** não contêm esse header → registrar no relatório, NÃO modificar

---

## Passo 4 — Loop de correção (máximo 3 tentativas)

Repetir até build passar ou 3 tentativas esgotadas:

1. Para cada arquivo do plugin com erros:
   - `Read` o arquivo completo
   - Analisar cada mensagem de erro associada
   - Aplicar correção com `Edit` (preserva o restante do arquivo)
   - Registrar: `<arquivo>: <descrição da correção>`

2. Rodar o build novamente (Passo 2)

**Tipos de erro e correção:**

| Erro | Correção |
|------|----------|
| `Property 'x' does not exist on type 'Y'` | Adicionar `x` à interface `Y`, ou corrigir o nome da propriedade referenciada |
| `Cannot find module '...'` | Corrigir path do import ou adicionar o import ausente |
| `Type 'string' is not assignable to type 'number'` | Ajustar o tipo na interface ou adicionar conversão `Number(valor)` |
| `Expected N arguments, but got M` | Ajustar a chamada para o número correto de argumentos |
| `Object is possibly 'undefined'` | Adicionar optional chaining `?.` ou verificação de nulidade |
| `Property 'x' has no initializer and is not definitely assigned` | Adicionar `!` ou inicializador ao campo |

**Não corrigir:**
- Arquivos sem header `@generated  poui-specialist`
- `angular.json`, `tsconfig.json`, `package.json`
- Erros que persistirem por 3 tentativas

---

## Passo 5 — Relatório final

**Build passou na tentativa N (N ≤ 3):**

```
## Build Verification

✅ Build passou na tentativa N — todos os erros foram corrigidos automaticamente.

   Correções aplicadas:
   - <arquivo>.component.ts: <descrição da correção>
```

**Erros restantes após 3 tentativas:**

```
## Build Verification

⚠ X erros corrigidos automaticamente nas tentativas 1-N.
❌ Y erro(s) restante(s) após 3 tentativas — requer(em) atenção manual:

   <caminho>:<linha>:<col>
   <mensagem do erro>
   → <dica de diagnóstico>
```

**Erros em arquivos pré-existentes (se houver):**

```
⚠ Erros em arquivos pré-existentes do projeto (não modificados pelo plugin):
   <caminho>:<linha> — <mensagem>
   Estes erros existiam antes da geração e precisam de correção manual.
```
